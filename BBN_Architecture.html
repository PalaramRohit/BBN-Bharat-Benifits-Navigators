3. Module Breakdown

🔍 RAG Retriever

Uses FAISS and SentenceTransformers. It recursively scans the Knowledge Base to perform semantic matching between user intent and policy text.

⚖️ Eligibility Agent

A deterministic rule engine that parses JSON policy criteria and compares them against user socio-economic attributes (age, income, location).

📈 Decision Engine

Aggregates agent outputs to provide "Success Probability" (ML) and "Benefit Maximization" (Ranking policies by financial value).

🤖 Explanation Agent

Gemini-powered LLM that removes "Bureaucratic Jargon" and provides a user-friendly, empathetic summary of rights and benefits.

4. Orchestrator Decision Logic

The BharatOrchestrator uses the following logical steps:

  • Intent Extraction: Detects if the user wants discovery, eligibility check, or application assistance.
  • Context Hydration: Enriches the query with the user's Profile & Document Status from MongoDB.
  • Parallel Selection: Routes the hydrated query to the RAG retriever and Eligibility agents concurrently.
  • Decision Synthesis: Synthesizes a unified response containing the Final Answer, Recommended Schemes, and the next Best Action.
