System Directory & Domain Boundaries

/backend/app/orchestration/orchestrator.py The Choreographer: Manages state hydration, async concurrency, and agent synthesis. It is the only component that talks to all others.
/backend/app/utils/rag_retriever.py The Knowledge Synapse: Handles vectorization of 90+ schemes. Uses all-MiniLM-L6-v2 for encoding and FAISS for sub-millisecond similarity search.
/backend/app/agents/eligibility_agent.py The Jurisprudence Engine: A low-latency deterministic engine that translates natural language criteria into hard Boolean logic trees.
/backend/app/decision/ The Analytics Core: Contains optimization_engine (benefit maximization) and document_advisor (gap analysis & readiness profiling).
/backend/app/utils/llm_client.py The Reasoning Interface: abstracts the Gemini 1.5 Flash API with advanced error handling and token-optimized prompt construction.

RAG Architecture: From Text to Vector

🔄 Data Ingestion Pipeline

Unstructured PDFs and semi-structured JSONs are passed through the bulk_processor.

  • Chunking: Policies are chunked by meaningful "Benefits" and "Eligibility" headers.
  • Embedding: Each chunk is mapped to a 384-dimensional vector space.

This is the "Math Layer" that happens between RAG and the final response.
